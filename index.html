Why AngularJS ?

  • HTML-like syntax means design is portable
  • Directives enable us to provide powerful tools for web designers
  • Familiar DI and MVC model for Spring developers
  • Demarcation of UI development and backend RESTful services provided by Spring
  • Testability

Effort vs Benefit

  • Learning Curve
  • AngularJs compare to jsp, jQuery, GWT?

Learning Curve

  • Very well documented (even has AngularJs song)
  • Fast to build HelloWorld application
  • Easy to learn basic concepts like controller, route, view, binding
  • Directive is easy to use (Andrew can use it)
  • If you want to pass objects around then need to pick up concepts like scope, rootscope,...
  • Need to learn Angular way of testing
  • Directive is hard to read and hard to write

General Pros & Cons

  • Medium sizes (angular.js < 1MB)
  • Modules size vary from 6kb to ~100kb (without obfuscation)
  • Easy to use alone but may take extra effort to integrate with third-party js libraries
  • The more directives you have, the smaller is the code base
  • Clean project structure
  • 2-way binding is good
  • Quite responsive

Compare with Jsp

  • Loose coupling is the way to go: Angular Webapp access Restful server (which can be Java or .NET)
  • Client-side scripting is a safer investment
  • Single page application provide better user experience
  • Developer and Designer can work together on the same source code
  • Directive may be equals to jsp tag + javascript
  • Can be viewed without server
  • May be harder to test if use HtmlUnit or WebUnit
  • May be harder to integrate with third-party javascript

Compare with jQuery + native Javascript

  • Scope is a plus
  • Pre-defined project structure
  • 2-way binding save a lot of typing
  • Smaller code base and less control
  • More difficult to integrate with third-pary library, it is better to have equivalent directive
  • Directive is easier to read and easier to track compare to registering event

Compare with Gwt

  • Readability
  • Ability to run without server
  • RPC is very tight-coupling
  • Possible to use Browser Debug
  • No more swearing and waiting for webapp deployment

Collaboration

  • Angularised page still look very similar to original template. Designer can work on real webapp
  • Angular integrate well wit Bootstrap and jQuery

Future Proof

  • Big Community
  • Angular module and directive acting like Apache Common
  • Angular 2, which built on ECMASCript 6 will not be backward compatible with older browser
  • Angular 1.x may be developed in parallel as Play framework 1 & 2
  • Angular 1.2 support IE8, Angular 1.3 support IE9
  • Directive protect the template from underlying technology (replace Bootstrap with something else?)
  • Client side script has longer life-span compare to server side scripting

Conclusion

On collaboration

  • Keeping design ahead
  • Practical tools are enough
  • Keep an open mind for change
  • Respect could go a long way

On workflow

  • Push improvements often
  • Wireframes are good visual aids to understand flow
  • AngularJS allows designers and developers to collaborate easily on projects
  • Using a UI framework like Bootstrap helps establish conventions on UI vocabulary
  • Developers learn how to layout UIs using Bootstrap as well
  • We are able to iteratively improve the design without impacting the amount of effort required
